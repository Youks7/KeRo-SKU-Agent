# 产品事实与保真规则

## 1. 识别输入与图像诊断

为每张输入图标记用途、视角、清晰度、曝光、色偏、反光、透视、遮挡、是否包含包装、Logo、标签、配件和变体。区分用户真实 SKU 图、竞品参考图、风格参考图和 AI 概念图。

## 2. 建立事实层级

- 已确认事实：图片清晰可见、用户明确提供或文件证明。
- 谨慎推断：用于策略判断但不能写成正式卖点。
- 禁止主张：缺少证据或属于高风险承诺。
- 缺失证据：会影响平台发布、素材真实性或用户选择的信息。

不要从视觉外观直接断言具体材质、成分、功率、容量、兼容性、功效、认证或产地。

## 3. 建立 IDENTITY_CONTRACT

记录不可变特征、有限可变特征、未知结构、允许变换、禁止变换、身份锚点和变体交付映射。对正面、侧面、背面、细节和包装分别记录 `view_confidence`；没有证据时不生成暴露隐藏结构的新机位。

身份锁定不等于锁死原图像素。商品的构图、场景、人物、光线、阴影、反射、材质环境、图形和叙事可以按槽位改变。

## 4. 选择 F0–F3

- F0：证据原图或忠实精修，用于严格身份/声明槽位。
- F1：真实产品像素与创意场景合成。
- F2：多视图、蒙版和身份锚点约束的身份条件编辑/重构；必须人工审核。
- F3：概念方向探索，不得直接发布。

按素材槽位决定，不用一个模式控制整页。素材槽位或平台禁止数字渲染时使用 F0；工具能力未知时仍可用 F1 生产创意场景底图。

## 5. 提炼视觉记忆点

每个记忆点都要包含：具体细节、证据位置、为什么属于本产品、适合的素材槽位。禁止把“高级、干净、科技、品质”单独作为记忆点。

## 6. 输出原则

先给关键结论，再给证据和风险。默认保持简洁，但不得省略事实边界、处理模式、禁止修改项和缺失证据。
