# KeRo SKU 公共安全契约

本文件是各平台 Skill 必须继承的最小安全规则。平台规则可以更严格，但不得降低这些要求。

## 事实来源

只把以下内容视为已确认事实：

1. 用户明确提供的产品资料。
2. 用户上传图片中清晰可见且不存在歧义的信息。
3. 用户提供的检测报告、认证、规格表、包装文件或品牌资产。
4. 已核对的目标平台后台类目规则。

把基于外观、类目经验或竞品观察得到的内容标记为“谨慎推断”。不要把推断写成正式卖点。

## 禁止编造

不得编造或补全品牌、型号、材质、成分、尺寸、重量、容量、性能、功效、适配范围、产地、认证、专利、销量、排名、评价、奖项、价格、库存、交付周期、产能、售后或平台身份。

无法确认具体材质时，只描述“可见质感”“表面呈现”或“视觉上接近”，并保留待确认状态。

## 产品处理模式

- 模式 A 严格保真：真实销售素材默认优先使用。保留真实产品本体，只生成背景、光影、场景和留白，再进行合成。
- 模式 B 参考图辅助：只能基于真实产品参考图进行有限编辑。不得改变 SKU 身份；主图、属性图或平台禁止数字渲染的槽位不得默认使用。
- 模式 C 概念生成：仅用于提案和视觉测试。必须明确标注不可作为真实销售 SKU 成品图。

## 产品身份保护

不得改变主颜色、比例、结构、数量、配件、包装、Logo、包装文字、标签位置或关键表面特征。不得把相似产品、竞品产品或模型虚构产品当作用户 SKU。

所有模式 B 输出都必须与真实图进行一致性检查；无法确认一致时降级为“仅供方向参考”或切换模式 A。

## 文字与品牌资产

默认不让图像模型生成最终标题、Logo、包装文字、参数表、认证标志或 CTA。使用真实品牌资产和独立文字图层。目标平台禁止图片覆盖文字时，遵守更严格的平台规则。

## 竞品参考

只参考页面节奏、信息层级、通用构图和光线逻辑。不得复制竞品产品、包装、Logo、文案、独特配色、认证标记或可识别活动视觉。

## 高风险类目

医疗、保健、美妆功效、食品、母婴、儿童用品、防护用品、电器、安防和宠物健康类目必须提高证据门槛。缺少证明时不生成正式功效、适用范围或安全承诺。

## 输出状态

每轮保留：已确认事实、谨慎推断、禁止主张、缺失证据、处理模式、已选方向、平台与站点、素材槽位、默认假设和下一步唯一决策。
