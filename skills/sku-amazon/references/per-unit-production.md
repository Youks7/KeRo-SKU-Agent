# 逐屏 / 逐模块生产协议

本协议恢复 Lite V1.2.1 的逐屏生产深度，同时保留 V1.3 的平台专用素材槽位。把“生产单元”理解为平台实际使用的一屏、一个详情模块、一个图片槽位、一个 A+ 模块或一个含媒体的 PDP Section；不要因此把所有平台强制做成固定长图。

## 触发与硬门槛

1. 只在用户确认方向后输出正式生产 Prompt。用户说“按你推荐的来”可视为确认，但必须列出默认假设。
2. 先确认平台、站点或市场、类目、素材槽位、目标语言和当前规则来源。规则不确定时标记 `human-review`，不要凭记忆伪造精确尺寸或限制。
3. 每个生产单元都引用 `SKU_CONTEXT`、`IDENTITY_CONTRACT` 和已批准 `CREATIVE_CONTEXT` 中的来源图片、身份锚点、创意命题、按槽位处理模式和禁止修改项，并记录目标工具是否支持多图参考、局部编辑和 Negative Prompt。
4. 缺少会影响商品身份、交付内容、声明或平台合规的证据时，把对应单元标记为 `blocked`，保留字段和占位符，不生成可误用为正式成品的 Prompt。
5. 任何价格、尺寸、材质、成分、性能、兼容性、认证、评价、排名、产能、MOQ、售后或促销声明都必须绑定证据来源。
6. 平台专用规则优先于本协议。平台禁止覆盖文字、数字渲染、场景背景或某种比例时，使用更严格要求。
7. 每个生产单元必须完整输出本协议字段，不得用一句“同上”省略 Prompt、Negative Prompt、处理模式、排版、镜头、质检或拦截自检。
8. 使用参考详情页迁移时，当前平台的 `reference_migration_context.platform_migrations[]` 必须已经批准；未批准时只允许输出模块映射和方向，不得进入正式生产。

## 整页生产总控

在逐单元内容之前先输出一次 `整页生产总控`：

- 已确认方向、推荐理由、目标平台、站点、类目、语言和页面/素材类型。
- 身份合同摘要、创意命题与处理模式总原则，以及每个槽位为何选择 F0、F1、F2 或 F3。
- 生产单元总表：编号、平台槽位、销售任务、证据状态、预计比例/响应式行为、发布状态。
- 信息故事线：识别商品 → 建立利益 → 提供产品证据 → 解释规格/使用 → 消除顾虑 → 信任/行动；按平台和商品删改，不机械套用。
- 整页视觉 Token：主辅色、背景材质、光源方向、对比度、字体层级、网格、边距、文字安全区、圆角/线条规则和禁用元素。
- 参考抽象：只记录可借鉴的构图、镜头、光线、材质、图形与节奏，以及明确排除的竞品商品、Logo、文案和独特符号。
- 参考迁移：记录 migration_id、M1/M2/M3、参考模块到目标单元的映射、允许迁移的抽象特征、明确排除元素和平台覆盖项；没有参考迁移时写 `N/A`。
- 跨屏连续性：产品朝向、镜头距离节奏、背景过渡、产品占比、文案密度和相邻单元衔接方式。
- 缺失证据和被阻塞单元，不得用默认值偷偷补齐。

## 生成工具路由

- 支持多图参考、结构控制或蒙版：允许在平台槽位许可且视角证据充分时使用 F2；明确身份合同、允许修改区、禁止修改区和人工审核闸门。
- 只支持单图参考编辑：不得使用 F2；改用已有真实机位的 F0/F1。任何需要重绘商品、新机位或佩戴关系的单元标记 `blocked`，直到多视图、蒙版或结构控制工具可用。F1 只能更换环境与合成条件，不能创造新的产品视角。
- 只支持文字生图：真实销售 SKU 使用 F1，生成完整场景底图并后期合成真实产品；严格证据槽位使用 F0。
- 不支持 Negative Prompt：把关键禁止项写入正向 Prompt、处理模式和质检；`Negative Prompt` 字段标记为“不支持”，不要伪造无效参数。
- 模型能力未知：先输出模型无关画面规范，不假定 F2 可用；仍可规划 F1 场景、光线、阴影和版式，不把整页退化为纯色换背景。
- 纯网页组件：Prompt 标记 N/A，价格、Variant、CTA 和政策使用真实数据与组件。

## Prompt 准确性规则

1. Prompt 开头明确输入依据、平台槽位和任务，不把策略说明伪装成视觉事实。
2. Prompt 只使用已确认事实；谨慎推断只能用于方向说明，不得进入正式商品声明。
3. F2/F3 的 Prompt 写出至少两个能回指来源的产品专属视觉细节；F1 背景 Prompt 不生成商品本体，产品专属信息写入真实图合成与质检字段。
4. F2/F3 明确产品数量、视角、位置、背景、光源、阴影、留白和文字安全区；F1 只描述背景构图、真实产品占位区、光源和合成条件。未确认尺寸不用虚构毫米数。
5. 生图模型不可靠地生成文字、Logo、包装标签、参数表、价格或认证标识时，Prompt 必须要求不生成，并交给后期真实图层处理。
6. 不把 Negative Prompt 当作事实保护的唯一手段；关键身份约束同时写入正向 Prompt、处理模式和质检。
7. 同一长页的 Prompt 共享视觉 Token，但每个单元必须改变商业任务或镜头证据，避免只换背景颜色。
8. 使用迁移蓝图时，Prompt 只能写入批准的抽象特征与真实 SKU 证据；不得把参考商品、人物、Logo、包装、原文案、专属符号或独特版式坐标写进生成控制。

## 产品处理模式

### F0｜证据原图

- 使用用户真实照片、真实裁切或忠实精修；不生成商品本体。
- 只允许裁切、清洁、曝光、白平衡和不改变事实的基础修复。
- 用于主图、SKU/variation、包装、数量、认证和声明证据等严格身份槽位。

### F1｜像素保留合成

- 产品本体使用真实抠图或真实局部图；生成模型负责完整的场景、光影、材质环境、道具、留白和接触阴影条件。
- 背景 Prompt 使用 `PRODUCT_CUTOUT_ZONE`，不生成商品类别、Logo、标签、包装或相似产品。
- 创意空间不局限于换纯色背景：允许建筑光影、叙事空间、材质世界、跨屏连续性和图形系统，只要不制造错误用途、尺度或功能暗示。

### F2｜身份条件编辑 / 重构

- 只在平台槽位允许且工具支持多图参考、蒙版或结构控制时使用。
- 使用 `IDENTITY_CONTRACT`、多视图、`view_confidence` 和身份锚点；新机位不得暴露无证据的隐藏结构。
- 正向 Prompt 明确保留颜色、轮廓、结构、比例、Logo、标签、包装文字和关键外观，只改变已批准的机位、人物、场景、光线或环境反射。
- Negative Prompt 禁止现有元素被新增、删除、替换、重绘或拼错；不得把裸词 `logo`、`text`、`label`、`product packaging` 一刀切列为负面词。
- 输出默认为 `human-review`；身份失败时返修或降级 F1/F0，不得自动标记 `ready`。

### F3｜概念探索

- 仅用于方向或极端视觉测试，必须标记 `direction-only`。
- 可以探索抽象表现，但不能引用不存在的检测、认证、性能或品牌事实。
- 不得作为真实售卖 SKU 成品，也不得在后续步骤自动升级为 `ready`。

处理模式按平台素材槽位决定。主图、SKU/variation、包装文字敏感和平台禁止数字渲染的槽位优先 F0；详情、附图、A+、PDP Section 或推广内容按规则开放 F1/F2。

## 文案与后期排版

1. 把最终文字、Logo、价格、参数、认证和 CTA 保留为可编辑真实图层或语义化 HTML，不依赖模型生成可读文字。
2. 写清标题、副标题、证据文案和脚注的具体位置、对齐、最大行数、文字安全区及移动端最小可读性。
3. 不使用“高端、品质、专业、黑科技”等空泛词替代商品证据。
4. Shopify 的价格、Variant、Add to Cart 和政策信息使用真实组件；Amazon、TikTok Shop 等禁止覆盖的槽位不添加文字。
5. 长图拼接前统一字体、字号层级、网格、边距、背景过渡和导出色彩空间；检查接缝、裁切和重复信息。

## 镜头矩阵与连续性

1. 为每个单元明确景别、角度、机位、产品状态、产品占比、背景、道具、光线、景深和透视。
2. 相邻单元至少改变镜头距离、视角、产品状态、证据重点或背景逻辑中的两项，但保持同一 SKU、光影世界和视觉 Token。
3. 全页至少包含识别镜头、产品专属细节镜头和购买确定性信息；缺少真实来源时阻塞对应镜头。
4. 不为了视觉节奏添加用户不会收到的配件、赠品、包装、场景能力或使用结果。

## 质检与通用 Prompt 拦截

逐单元检查后，再执行整页检查：

- 商品身份、颜色、结构、变体、数量和包含物是否从头到尾一致。
- 页面顺序是否符合目标平台的购买任务，而非固定六屏模板。
- 同一卖点、相似构图或同一视角是否无意义重复。
- 所有数字、材质、性能、认证、比较、价格和服务承诺是否可追溯。
- 文字是否可读、可编辑、未被模型误生成，移动端是否保留安全区。
- 平台主图、SKU 图、详情模块、广告素材、A+ 或 PDP Section 是否被错误混用。
- 迁移单元是否绑定真实证据、遵守 M1/M2/M3 权利边界、排除参考页识别性元素，并按目标平台重新映射。
- `blocked` 和 `direction-only` 单元是否被误标为可发布。

## 输出节奏

- 用户未要求一次性输出全部时，先完整输出前 3 个生产单元，确认视觉系统后再继续。
- 用户明确要求完整详情页时，一次性输出所有单元，但每个单元都必须使用完整模板。
- 用户只要求首屏时，只输出首屏的完整模板。
- 用户要求首屏 A/B 测试时，输出 2 至 3 个完整首屏方案；每个方案分别执行拦截自检。
- 用户只要求策略或方向时，不提前输出正式生产 Prompt。
