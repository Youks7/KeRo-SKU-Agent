# 淘宝平台规则与输出结构

> 规则核对日期：2026-07-12。实际发布以淘宝商家后台返回的类目 Schema 和尺寸校验为准。

## 必要输入

- 淘宝市场、叶子类目、商品类型和销售地区。
- 主图、轮播、SKU 属性图或详情模块中的具体交付槽位。
- 所有颜色、尺码、规格、套装、赠品和价格关系。
- 类目资质、品牌授权和平台后台当前提示。

## 素材槽位

### 搜索与主图媒体

淘宝发布端区分 1:1 和 3:4 主图槽位。不要把纵向详情长图直接当主图。输出前记录用户后台当前允许的比例、数量、格式和覆盖元素规则。

主图优先解决：商品身份、实际售卖内容、关键差异和点击后的预期一致。不得用高配 SKU、完整套装或赠品画面代表最低价非同款 SKU。

### 轮播图

按商品身份、角度、核心利益、使用、细节、尺寸/规格和疑虑处理排序。避免重复同角度；平台或类目禁止文字覆盖时使用纯图。

### SKU 属性图

每张图只代表对应变体。颜色、图案、数量和包装必须与属性值一致。建立 `attribute_value -> source_image -> approved_asset` 映射。

### 详情模块

移动端优先，按独立销售任务拆分模块。比例按发布工具和设计系统确定，不默认所有模块都是 9:16。参数和规格使用可编辑文字层。

## 方向生成

根据类目生成实际商业任务，例如快速识别、核心差异、使用证明、尺寸确定性、礼赠或疑虑消除。方向必须改变首图构图、信息顺序和证据重点，而不只是换色调。

## 一致性检查

- 标题、类目、属性、主图、SKU 图和详情描述是否一致。
- 赠品、套装、数量和最低价 SKU 是否造成误导。
- 材质、功效、适配、销量和认证是否有证据。
- 详情图片文字是否能在移动端清晰阅读。

## 输出结构

```text
当前规则来源与日期
商品和SKU摘要
素材槽位清单
SKU属性图映射
方向提案
确认后的逐槽位方案
事实与描述不符风险
发布检查
```
