# 产品事实核心质检

- 图片与资料是否来自同一个 SKU。
- 可见事实、用户陈述、文件证据与谨慎推断是否分开。
- 是否编造材质、规格、功效、认证、评价、排名、价格或售后。
- 是否建立 `IDENTITY_CONTRACT`，并分开不可变、有限可变和未知特征。
- F0–F3 是否按平台素材槽位选择，而不是用一个模式压制整页。
- 禁止修改项是否覆盖颜色、结构、比例、Logo、包装、标签和关键细节。
- 每个视觉记忆点是否包含具体细节、证据位置和适用槽位。
- 多视图是否属于同一 SKU；每个新机位是否有足够 `view_confidence`。
- 是否建立 `CREATIVE_CONTEXT`、参考抽象报告和三个差异化方向。
- 任意两个方向是否至少在创意命题、场景世界、商品角色、镜头与光线、叙事节奏中的三个维度不同。
- 方向是否只换色、道具或通用“高级感”；视觉隐喻是否暗示未验证性能。
- `SKU_CONTEXT` 未知字段是否保持为空，默认值是否进入 `confirmed_assumptions`。
- 新证据升级事实时是否记录来源，平台切换时是否保留产品身份。
- 是否只提出真正会改变真实性、平台路线或生产模式的问题。
- 恢复项目时是否从已保存阶段继续，而没有重复产品分析、参考分段或方向生成。
