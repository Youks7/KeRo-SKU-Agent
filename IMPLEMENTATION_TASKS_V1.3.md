## 完成定义

V1.3.0 只有在满足以下条件时才算完成：

- [ ] 保留 `sku-detail-page-director` 作为旧用户兼容入口和总路由。
- [ ] 建立统一的产品事实、保真边界和 `SKU_CONTEXT`。
- [ ] 淘宝、天猫、拼多多、京东、1688、Amazon、Shopify、TikTok Shop 均有独立 Skill。
- [ ] 单平台请求只加载对应平台规则，不加载无关平台文件。
- [ ] 多平台请求只分析一次产品事实，再分别生成平台原生方案。
- [ ] 每个 Skill 可单独安装，并包含必要的公共安全底线。
- [ ] 所有 Skill 通过 frontmatter、命名、引用路径和安装包一致性验证。
- [ ] 每个平台至少有一个完整测试案例和一个失败纠正案例。
- [ ] 同一个测试 SKU 在不同平台输出的素材槽位、页面结构和销售逻辑明显不同。
