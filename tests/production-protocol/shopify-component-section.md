# Shopify 真实组件协议回归样例

## 整页生产总控

目标平台：Shopify PDP。该样例验证逐单元协议不会把真实页面组件烘焙进固定长图。

## 生产单元 04｜购买组件

平台槽位 / 模块类型：Product information / Add to Cart section
销售任务：让用户选择真实 Variant 并完成加购。
发布状态：blocked
本单元依赖：真实 Variant、价格、库存和销售政策数据。
禁止主张 / 缺失证据：当前缺少 Variant、价格、库存、配送和退换货数据。

创意命题：用真实可交互购买组件承接媒体叙事，不制作图片内假按钮。
槽位创意自由度：0——数据和交互完全锁定，只有主题视觉样式可调整。
身份锚点：Variant、价格、库存、商品媒体和实际包含物。
参考抽象：只参考响应式信息层级，不复制其他店铺组件或品牌资产。
发布闸门：真实商品数据接入、可访问性和 Variant 映射通过后才能 `ready`。

未来画面：主题原生产品信息区域；不生成带假价格或假按钮的图片。
主文案：使用真实商品标题字段。
副文案 / 证据文案：使用已核对的价格、Variant、配送和退换货字段。
文案位置与安全区：由主题响应式网格控制，移动端 CTA 保持可触达且不遮挡内容。

产品处理模式：F0
处理理由：使用真实商品媒体和真实交互组件，不生成产品或按钮。
生成工具能力：N/A，使用语义化 HTML 与真实主题组件
允许修改：主题样式、间距、对比度和合法文案。
禁止修改：真实价格、Variant、库存、商品身份和政策。
产品来源与合成方式：使用 Shopify 商品媒体与语义化 HTML 组件。

镜头矩阵：
- 景别：N/A，组件单元。
- 视角 / 机位：N/A，组件单元。
- 产品朝向 / 状态：由相邻真实 Gallery 媒体提供。
- 产品占比 / 位置：桌面双栏、移动端单栏，由主题断点控制。
- 背景 / 道具：主题背景，无生成道具。
- 光线 / 阴影 / 反射：由真实媒体单元控制。
- 景深 / 透视：N/A，组件单元。
- 与前后单元的差异及衔接：承接 Gallery，下一单元进入已验证利益和细节。

Prompt：N/A——本单元使用语义化 HTML / 真实组件，不生成图片文字或假按钮。
Negative Prompt：fake Add to Cart button, fake price, fake discount, fake inventory, text baked into image, inaccessible image-only controls

后期排版：
- 画布比例 / 尺寸 / 响应式裁切：使用主题断点，不输出固定长图尺寸。
- 文字图层、字体层级、对齐和行数：语义化标题、价格、Variant label 和按钮文本。
- 产品、文字、标识和 CTA 的安全区：CTA 为真实可聚焦组件，移动端避免被 sticky UI 遮挡。
- 色彩、对比度、边缘、阴影和导出要求：满足对比度、键盘操作和性能要求，不导出静态按钮图。

产品一致性质检：
1. Variant 切换必须对应真实 SKU 媒体。
2. 价格、库存和包含物必须来自商品数据。
3. 不得使用图片内假 CTA、假评价或假折扣。
4. 缺少商业字段时保持 `blocked`。
5. 数据接入并人工核对后再改为 `ready`。

通用 Prompt 拦截自检：
1. 已明确本单元不应生成图片 Prompt。
2. 未把 Shopify PDP 错做成固定六屏或八屏长图。
3. 未混入缺失价格、Variant、库存或政策。
4. 与相邻 Gallery 媒体职责不重复。
5. 结果：通过。
